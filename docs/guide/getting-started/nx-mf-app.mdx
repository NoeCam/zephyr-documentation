## Generating a new Nx workspace
One of the most popular Monorepo and CI tools [Nx](https://nx.dev) includes a series of [generators](https://nx.dev/recipes/module-federation) for creating federated applications.
This example starts by creating an empty Nx workspace.

```bash filename="Terminal"
npx create-nx-workspace@latest myorg
```
If you want to use Rspack you can use this command instead.
```bash filename="Terminal"
npx create-nx-workspace@latest myorg --preset=@nx/rspack
```

### Frontend Framework
At this time we do not want to select a frontend framework so select `None`.
```bash filename="Terminal"

```

### Monorepo Type
We prefer to use `Integrated` since we will have all of the remotes and shell in one repo.

### Nx Cloud
Any option works for Nx Cloud. Our example chooses `Skip for now` however we encourage you to use Nx Cloud as it is very helpful for speeding up local and CI builds.

## Next Steps
CD into the folder that was generated.
```bash filename="Terminal"
cd myorg
```

## Adding Framework / Library Support
At this point we will use Nx to add React support. You can use another framework if you want but our example uses React.
```bash filename="Terminal"
npx nx add @nx/react
```

## Installing Zephyr
Install the `zephyr-edge-contract` and `zephyr-webpack-plugin` by copying them from the Zephyr examples [repo](https://github.com/ZephyrCloudIO/zephyr-examples/tree/main/_zephyr_pack).

:::warning
In the future these two packages will be available via NPM however during Alpha 1 testing they are located in the Zephyr Examples [repo](https://github.com/ZephyrCloudIO/zephyr-examples/tree/main/_zephyr_pack)
:::

Add both Zephyr packages as a dependency to your `package.json`.
The file path for our examples repo is different than the one below due to the examples repo sharing the packages across multiple examples.
```json filename="package.json"
"zephyr-edge-contract": "file:_zephyr_pack\\zephyr-edge-contract-0.0.2.tgz",
"zephyr-webpack-plugin": "file:_zephyr_pack\\zephyr-webpack-plugin-0.0.2.tgz"
```

## Generate Module Federation Shell and Remotes
Use Nx to generate the host and shell applications using the below commands.
```bash filename="Terminal"
npx nx g @nx/react:host shell --remotes=remote1,remote2 --directory=apps/shell
```

If you want to use Rspack you can use this command instead.
```bash filename="Terminal"
npx nx g @nx/react:host shell --remotes=remote1,remote2 --directory=apps/shell --preset=@nx/rspack
```

## Modifications
We now will need to make some changes to the generated projects to add Zephyr Cloud support.

## Adding Package JSON Files
Create Package.json files in each app. Zephyr Cloud currently requires a
package json in order to resolve names. You will need to create a `package.json` in each of the applications with a name.
```bash filename="myorg/shell/package.json"
{
"name": "shell"
}
```
```bash filename="myorg/remote1/package.json"
{
"name": "remote1"
}
```
```bash filename="myorg/remote2/package.json"
{
"name": "remote2"
}
```

## Updating Webpack Config
Update both the `webpack.config.ts` and `webpack.config.prod.ts` files for shell and `webpack.config.ts` for both remotes.

:::info
[Rspack](https://rspack.dev/) is also supported however this example uses webpack
:::

```ts filename="shell/webpack.config.ts"
import { composePlugins, withNx, ModuleFederationConfig } from '@nx/webpack';
import { withReact } from '@nx/react';
import { withModuleFederation } from '@nx/react/module-federation';

import baseConfig from './module-federation.config';
import {withZephyr} from "zephyr-webpack-plugin";

const config: ModuleFederationConfig = {
  ...baseConfig,
};

// Nx plugins for webpack to build config object from Nx options and context.
export default composePlugins(
  withNx(),
  withReact(),
  withModuleFederation(config),
  withZephyr(),
  (config)=>{
    return config;
  }
);
```
```ts filename="shell/webpack.config.prod.ts"
import { composePlugins, withNx } from '@nx/webpack';
import { withReact } from '@nx/react';
import { withModuleFederation } from '@nx/react/module-federation';
import { ModuleFederationConfig } from '@nx/webpack';

import baseConfig from './module-federation.config';
import {withZephyr} from "zephyr-webpack-plugin";

const prodConfig: ModuleFederationConfig = {
  ...baseConfig,
  /*
   * Remote overrides for production.
   * Each entry is a pair of a unique name and the URL where it is deployed.
   *
   * e.g.
   * remotes: [
   *   ['app1', 'http://app1.example.com'],
   *   ['app2', 'http://app2.example.com'],
   * ]
   *
   * You can also use a full path to the remoteEntry.js file if desired.
   *
   * remotes: [
   *   ['app1', 'http://example.com/path/to/app1/remoteEntry.js'],
   *   ['app2', 'http://example.com/path/to/app2/remoteEntry.js'],
   * ]
   */
  remotes: [
    ['remote1', 'http://localhost:4201/'],
    ['remote2', 'http://localhost:4202/'],
  ],
};

// Nx plugins for webpack to build config object from Nx options and context.
export default composePlugins(
  withNx(),
  withReact(),
  withModuleFederation(prodConfig),
  withZephyr(),
  (config)=>{
    return config;
  }
);
```

